## [2.7.0] — 2026-04-22

### Changed
- **Folder convention mới**: tách `docs/specs/` (BA/Dev) vs `docs/tests/` (QA/Tester) thay cho cây cũ `docs/features/` trộn lẫn.
- **Zone-Module-Feature hierarchy** chuẩn hóa: `{zone}/{module}/{Feature_ID}_{tên}/`.
- **M1/M5/M6 output paths** sang convention mới (per-screen specs cho M1, per-screen test cases cho M6).

### Added
- Template `test-execution-template.md` enrich theo chuẩn TC.IMS (+ Data Prerequisites, Cross-Module Impact, Test Matrix, Regression Checklist, Evidence section).
- Script `migrate-split-specs-tests.sh` auto-migrate project cũ (`docs/features/` → `docs/specs/` + `docs/tests/`) bằng `git mv`.
- `ui-mockup-patterns.md` **Stage 0: Pre-Read + Pre-Questionnaire** (Mode 10 không còn code ngay, phải đọc 8 nguồn + hỏi 5-15 câu trước).
- Section 0.5 **Multi-Skill Orchestration** — đề xuất activate skill bổ trợ theo context (mobile-design, kpi-dashboard-design, shadcn, ux-flow…).
- Section 7 **Post-Mockup Validation** — gợi ý chạy `ui-review` + `ux-audit` + `ui-a11y` + `uxui-principles` sau khi mockup render OK.
- Flow 8-step bắt buộc khi nhận trigger "Tạo mockup".
